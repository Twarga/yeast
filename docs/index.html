TwargaOps Local VM Engine

Local VMs, baked fast.

Yeast turns a project folder into real QEMU/KVM machines. Cloud-init, provisioning, snapshots, private lab networks, and clean JSON output — all from one config file.

Why Yeast

No more VM plumbing every time.

Running local VMs is still harder than it should be. You download cloud images, create qcow2 disks, write cloud-init files, build seed ISOs, compose QEMU arguments, track SSH ports, and manage state by hand. Yeast replaces that with a project workflow: define machines in yeast.yaml, run yeast up, and get real VMs.

The Loop

One config. One command. Real machines.

init pull up provision snapshot ssh down destroy
For builders

Project-native machines.

Put yeast.yaml in a folder and let that folder own the VM lifecycle. Trusted base images, automatic SSH, and clean state.

For labs

Real multi-VM labs.

Define attackers, targets, and monitors on a private network. Snapshot a clean baseline. Restore after breakage. Repeat.

For tools

Built for automation.

Humans get readable terminal output. Tools get stable JSON. Guest control commands let agents inspect and operate inside VMs.

In action

See it work.

Start a two-VM lab with a private network, check status, take a snapshot, and run commands inside the guest — all without leaving your terminal.

bash 
$ yeast up

$ yeast status
web    running  ssh:2222   lab:10.10.10.10
target running  ssh:2223   lab:10.10.10.20

$ yeast exec web -- uname -a
✓ Linux web-lab 6.8.0-31-generic

$ yeast snapshot web clean
✓ Snapshot saved: web/clean
