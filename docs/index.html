01 — Configuration

One file. All your machines.

Describe what you want. Yeast handles images, disks, cloud-init, networking, and state.

  • Project-scoped instances with unique IDs
  • Trusted base images in a shared cache
  • Automatic SSH key injection and port forwarding
  • Static lab IPs for multi-VM topologies
yeast.yaml 
version: 1
networks:
  - name: lab
    cidr: 10.10.10.0/24
instances:
  - name: web
    image: ubuntu-24.04
    memory: 1024
    cpus: 1
    ssh_port: 2222
    networks:
      - name: lab
        ipv4: 10.10.10.10
  - name: target
    image: ubuntu-24.04
    memory: 1024
    networks:
      - name: lab
        ipv4: 10.10.10.20
02 — Capabilities

Everything you need for local VMs.

01

Provisioning

Install packages, copy files, run shell commands. Automated post-boot setup over SSH.

02

Snapshots

Save a clean baseline. Break things. Restore in seconds. Labs are meant to be destroyed.

03

Lab Networks

Private project-level networks with static IPv4. Attacker, target, monitor topologies just work.

04

Guest Control

Exec, copy, logs, inspect — all from the host. Built for scripts, agents, and AI workflows.

03 — In Action

From zero to running VMs.

bash 
$ yeast init --template two-vm-lab
✓ Created yeast.yaml and .yeast/project.json

$ yeast pull ubuntu-24.04
✓ Image cached: ~/.yeast/cache/images/ubuntu-24.04

$ yeast up
✓ Started web (127.0.0.1:2222)
✓ Started target (127.0.0.1:2223)

$ yeast status
web    running  ssh:2222   lab:10.10.10.10
target running  ssh:2223   lab:10.10.10.20

$ yeast exec target -- uname -a
✓ Linux target-lab 6.8.0-31-generic x86_64

$ yeast snapshot target baseline
✓ Snapshot saved: target/baseline

$ yeast down
✓ Stopped 2 instances
