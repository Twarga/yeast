How it works

From zero to running VMs.

Yeast keeps the flow short: describe the lab, start it, then work inside real Linux machines.

1

Write yeast.yaml

Declare images, CPU, memory, disks, networks, and provisioning in the project folder.

2

Run yeast up

Yeast prepares the VM, downloads the image if needed, and boots it on QEMU/KVM.

3

Provision on boot

Cloud-init configures users, keys, packages, files, and first-boot setup.

4

SSH and build

Connect to real Linux machines and use them for labs, testing, automation, and learning.

Features

Built for local infrastructure labs.

The feature set is focused on repeatable VM environments, not fake shells or container-only demos.

YAML

Project configuration

Keep VM definitions in version control and rebuild the same lab from the same folder.

VM

Real virtualization

Run full Linux machines through QEMU/KVM with systemd, kernels, disks, and isolation.

NET

Private networking

Create multi-VM labs with predictable networking for app, database, proxy, and cluster setups.

INIT

Cloud-init provisioning

Bootstrap users, SSH keys, packages, commands, and files automatically on first boot.

SNAP

Snapshots and restore

Save known-good states, test risky changes, and return to a clean machine quickly.

JSON

Automation output

Use machine-readable output for scripts, checks, agents, CI jobs, and repeatable workflows.
