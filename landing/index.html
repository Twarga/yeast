Yeast orchestrates real QEMU/KVM virtual machines from a simple folder. You get isolated environments with full Linux userspace, systemd, cloud-init, and everything you expect from real machines.

How it works

From zero to running VMs.

1
yeast.yaml
Define your lab
2
yeast up
Boot real VMs
3
Running VMs
SSH-ready machines
4
yeast ssh
Connect and work
Key features

Everything you need for local VMs.

📝

YAML-driven labs

One file defines the whole project. Keep infrastructure in version control and avoid GUI drift.

VM

Real VMs

Full hardware virtualization with QEMU/KVM. Real kernels, real isolation, real Linux userspace.

💾

Snapshots

Stop, save, restore, and clone VMs in seconds. Reset safely without rebuilding everything from scratch.

🌐

Private networking

Lab VMs talk to each other on isolated networks with static IPs and predictable topology.

⚙

Cloud-init provisioning

Packages, users, files, and setup run automatically on first boot. No manual babysitting.

📊

JSON automation

Stable machine-readable output and events make Yeast easy to script and integrate.

Quick example

A folder becomes a running machine.

terminal project setup
$ mkdir my-lab && cd my-lab
$ yeast init
Project initialized.
terminal runtime
$ yeast up
Creating VM web-01 ...
Provisioning with cloud-init ...
web-01 is up and reachable via SSH
