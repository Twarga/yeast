What It Is

Not a cloud platform. Not containers.
Real VMs.

▶

Define machines in yeast.yaml

One YAML file describes your entire lab. Names, images, resources, networking, provisioning. Version control your infrastructure.

⚡

Run yeast up

One command boots real QEMU/KVM virtual machines with cloud-init provisioning. No GUI. No clicks. No hidden steps.

🔐

Get SSH-ready machines

VMs are reachable by SSH in seconds. Provisioned, configured, and ready to work. Built for developers who want real Linux.

Features

Everything you need for local VMs.

📝

YAML-Driven

One yeast.yaml defines your entire lab. Version control your infrastructure. No GUI needed.

🖥

Real VMs

Full hardware virtualization with QEMU/KVM. Not containers. Real Linux machines with real kernels.

💾

Snapshots

Stop VMs, take snapshots, restore to any state. Safe lab reset in seconds. Provisioned state preserved.

🌐

Private Networking

Private lab networks for multi-VM environments. Static IPs. VM-to-VM communication out of the box.

⚙

Cloud-Init Provisioning

VMs are provisioned with cloud-init on first boot. Packages, files, shell commands. No manual setup.

📊

JSON Automation

Stable JSON output and event streams for scripting. Integrate with LabsBackery, MCP, or your own tools.
