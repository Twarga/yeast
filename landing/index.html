How it works

From folder to running VMs.

Yeast keeps the workflow small: describe the machines, start the lab, then work inside real Linux guests.

01

Write yeast.yaml

Declare images, CPU, memory, disks, users, networking, and provisioning in the project folder.

02

Run yeast up

Yeast prepares images, disks, cloud-init seed data, and QEMU/KVM runtime files.

03

Provision on boot

Cloud-init and SSH-backed provisioning configure packages, files, and shell setup.

04

SSH and build

Connect to full Linux machines for labs, automation, testing, and repeatable learning.

Features

Built for local infrastructure labs.

The feature set stays focused on repeatable VM environments, not fake shells or container-only demos.

YAML

Project configuration

Keep VM definitions in version control and rebuild the same lab from the same folder.

VM

Real virtualization

Run full Linux machines through QEMU/KVM with systemd, kernels, disks, and isolation.

NET

Private networking

Create multi-VM labs with predictable networking for app, database, proxy, and cluster setups.

INIT

Cloud-init provisioning

Bootstrap users, SSH keys, packages, commands, and files automatically on first boot.

SNAP

Snapshots and restore

Save known-good states, test risky changes, and return to a clean machine quickly.

JSON

Automation output

Use machine-readable output for scripts, checks, agents, CI jobs, and repeatable workflows.
