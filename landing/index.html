How it works

From folder to running VMs.

Yeast keeps the workflow minimal. Describe the machines, start the lab, then work inside real Linux guests.

01

Write yeast.yaml

Declare images, CPU, memory, disks, users, networking, and provisioning — all in a single project file.

02

Run yeast up

Yeast prepares images, disks, cloud-init seeds, and QEMU/KVM runtime files. One command, everything boots.

03

Provision on boot

Cloud-init and SSH-backed provisioning configure packages, files, and shell setup automatically on first boot.

04

SSH and build

Connect to full Linux machines. Run labs, automation, tests, and repeatable workflows on real infrastructure.

Features

Built for local infrastructure labs.

Focused on repeatable VM environments — not fake shells, not containers masquerading as VMs.

YAML

Project configuration

Keep VM definitions in version control and rebuild the exact same lab from the same folder, every time.

KVM

Real virtualization

Full Linux machines through QEMU/KVM — systemd, real kernels, isolated disks, network namespaces.

NET

Private networking

Multi-VM labs with predictable IPs for app, database, proxy, and cluster topologies out of the box.

INIT

Cloud-init provisioning

Bootstrap users, SSH keys, packages, commands, and files automatically on first boot — zero manual steps.

SNAP

Snapshots and restore

Save known-good states, test risky changes, and return to a clean machine in seconds.

JSON

Automation output

Machine-readable output for scripts, CI jobs, agents, and repeatable workflows — a stable JSON contract.
